## Definition

The artifact pipeline is the progressive generation chain at the heart of forge. Each artifact type builds on the previous one, using the wiki as the shared knowledge layer. Artifacts are generated at each level of the [[concepts/work-item-hierarchy|work item hierarchy]] and propagate constraints and context downward.

## How it works

The pipeline produces artifact types per hierarchy level:

**Strategic** (Theme / Initiative):
- `brief` → `artifacts/brief.md`
- `requirements` → `artifacts/requirements.md` (quality attributes & constraints)
- `adr` → `artifacts/adr/NNN-title.md` + index
- `diagram` → `artifacts/diagrams/<type>.md`

**Product** (Epic / Feature):
- `requirements` → `artifacts/requirements.md` (functional requirements)
- `feature-list` → `artifacts/feature-list.md` *(Epic level only)*
- `feature-detail` → `artifacts/feature-detail/F-NNN-<slug>.md` *(Feature level only — requires feature-list in parent Epic)*
- `adr` → `artifacts/adr/NNN-title.md` + index
- `der` → `artifacts/der.md` (Entity-Relationship Diagram)
- `diagram` → `artifacts/diagrams/<type>.md`

**Tactical** (User Story / Task / Bug):
- `user-story` → `artifacts/user-story.md`
- `diagram` → `artifacts/diagrams/<type>.md`

Recommended generation order within a work item:
```
brief → requirements → feature-list → adr → der → diagram → user-story → diagram (seq/state)
```

`feature-list` is a **hard dependency** for `user-story` and for `feature-detail` — the skill will not proceed without it. `feature-detail` is a soft dependency for `user-story`: it enriches generation if present but is not required.

**Dependency enforcement** is strict: same-level dependencies are enforced (Claude stops and specifies what to generate first); cross-level hard dependencies (`feature-list` before `feature-detail` or `user-story`) are also enforced.

**Gap handling**: Claude never invents content from training knowledge. Any section with no wiki coverage gets a `> [!gap]` callout. Gaps are signals to ingest more sources or consult stakeholders.

Each artifact is generated by `/draft` using the wiki pages accumulated by `/ingest`. The Epic's `feature-list` feeds the Feature's `feature-detail`, which feeds individual `user-story` files.

Strategic-level artifacts propagate as constraints into product-level work via [[concepts/cross-work-item-context|cross-work-item context]] — they are read on demand from the parent's `output/artifacts/` folder, not copied.

## Evidence and claims

From [[sources/readme]]:
> "Every artifact traces back to a source document. Every decision can be explained."

From [[sources/research]]:
> "Each artifact that exists enriches the next one. `feature-list` is a hard dependency for `user-story`; all others are soft (enrich if present, not required)."

The progressive chaining model is the tooling realization of [[entities/protocolo-es-ai]], which defined a protocol for AI-enabled activities across the software development lifecycle.
